# OpenAllay

[English](README.md)

OpenAllay 是一个现代化 Minecraft Agent，专为整合包而生。你只需自然地提问，
它就能查看当前游戏、搜索配方与指南、检查背包，并把真正有用的信息整理成一份
清楚的回答。

你可以自由选择使用的模型。OpenAllay 也希望与玩家、整合包和社区创作的知识
与扩展一起成长。

![OpenAllay 将 Skills、Extensions、共享模型、丰富界面与视觉引导带入 Minecraft](docs/media/openallay-banner.png)

*自然地提问、跟随处理进度，不离开 Minecraft 就能探索丰富的答案。*

## 为什么选择 OpenAllay？

### 根据你的游戏来回答

OpenAllay 不只是生成一段带有 Minecraft 风格的文字。它能查看当前实例中的模组
与设置，搜索配方和指南书，还能把配方需求与你背包中现有的物品进行比较。遇到
更复杂的问题时，它会先完成多个步骤，再给出回答。

### 原生游戏内体验

回答可以包含物品图标、材料槽位、配方布局、表格、进度步骤和可展开的详情。
OpenAllay 在准备回答时仍会保持界面响应；即使暂时关闭界面，请求也不会因此
取消。

### 模型由你选择

你可以连接兼容 OpenAI Chat Completions 或 Anthropic Messages 的服务，保存多个
模型配置，并直接在对话中切换。OpenAllay 既能用于单人游戏，也能在没有安装
本模组的普通多人服务器上使用；安装了 OpenAllay 的服务器还可以为玩家提供
共享模型。

### 随时可以继续的对话

把不同话题放进不同会话，回看保存的对话，复制有用的消息，或者把整个会话导出
留存。Skills 会为特定玩法、模组和问题类型提供额外指导，你也可以直接在游戏中
查看和管理它们。

## 现在可以做什么

- 查找配方与物品用途，并检查背包里是否有需要的材料。
- 查看已安装的模组、视频设置、资源包、坐标和 F3 风格的信息。
- 搜索支持的指南书内容和本地笔记。
- 在处理多步骤问题时查看实时进度和清晰易懂的工具详情。
- 在 OpenAllay 原生界面中使用多个会话、历史记录、模型配置、复制与导出。
- 使用游戏本身和兼容配方查看器提供的配方信息，包括 JEI 与 REI；即使没有安装
  它们，OpenAllay 的基础配方功能仍然可用。

## 快速上手

请从 Modrinth 下载 OpenAllay 的 **Fabric** 或 **NeoForge** 快照版本。当前版本
面向 Minecraft **26.2**，需要 Java **25**；Fabric 玩家还需要安装对应版本的
Fabric API。

把下载的 JAR 放入游戏实例的 `mods` 文件夹，启动 Minecraft，然后连接模型：

1. 进入一个世界，按 **K**，或输入 `/guide`。
2. 点击齿轮按钮，打开**模型**页面，再添加一个模型配置。
3. 选择 **OpenAI 兼容 Chat Completions** 或 **Anthropic Messages**。
4. 填写服务地址、模型 ID、上下文窗口和 API 密钥，然后保存。
5. 选择刚刚添加的配置，开始提问。

在 Fabric 上，Architectury **21.0.2 及更早版本**会导致 OpenAllay 界面无法
输入文字。请升级到 **21.0.4 或更新版本**，或移除 Architectury。

## 使用游戏内界面

OpenAllay 会在一个不暂停游戏的 Minecraft 原生界面中打开。状态栏会显示它正在
做什么，以及当前请求已经运行了多久。

- 按 **Enter** 发送消息；按 **Shift+Enter** 换行。
- **停止**会取消当前请求；**重试**会重新发起请求。
- 按 **Escape** 只会关闭界面。再次打开后，仍能看到继续生成的回答。
- 可以用不同会话区分不同话题，也可以随时切换到另一个已配置的模型。

## 模组与内容支持

OpenAllay 会利用当前整合包中已经存在的内容：

- **JEI** 配方可以在 OpenAllay 中使用熟悉的 JEI 布局。
- **REI** 可以为 OpenAllay 的配方体验提供配方信息。
- 可以直接在游戏中搜索当前资源里的 **Patchouli** 指南书内容。
- **Farmer's Delight** 等配方丰富的模组可以使用配方搜索、材料检查和可视化
  配方页面。

这些集成都不是必需项。即使其中某项没有安装，或不适用于当前模组环境，
OpenAllay 仍然可以继续使用。

## OpenAllay 的未来方向

OpenAllay 正在朝开放平台的方向发展。下面列出的是路线图，不代表当前快照已经
包含这些完整能力：

- **OpenAllay Skills** 会变得更容易由玩家与社区创作、改进、分享和发现。
- **OpenAllay Extensions** 计划支持社区制作的工具、知识来源、模组集成与游戏
  原生界面组件。
- **OpenAllay Host** 是计划中的服务器体验，用于共享模型和统一管理的伙伴
  服务。
- **OpenAllay Studio** 是未来的创作工具，用来制作更丰富的游戏内内容、动态
  界面与可重复使用的引导体验。

### 接下来

- 可以由玩家查看、纠正、固定或遗忘的玩家记忆。
- 更顺畅的社区 Skills 创作与分享流程。
- 接入更多模组、指南与 Minecraft 内容来源。
- 为社区 Extensions 提供第一批受支持的接入方式。

### 更长远的方向

- 理解地图、附近环境、结构、方块与容器。
- 把结构与文档变成分步骤的视觉教程，并在有兼容集成时提供 Ponder 风格引导。
- 跨机器、中间材料与大型科技树规划完整生产链。
- 持续发展 OpenAllay Host、Studio 与更广泛的 Extensions 生态。

OpenAllay 是独立项目，与 Mojang Studios 或 Microsoft 没有从属关系，也未获得
其认可。
